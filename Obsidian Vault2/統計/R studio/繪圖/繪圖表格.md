# 設定參數 (純量控制)、範例
dodge_w <- 0.9  # 分組間距 (對齊用)
bar_w   <- 0.7  # Bar 寬度 (外觀用)
star_gap <- max(df$mean)*0.05 # 星星懸浮高度

ggplot(df, aes(x = Time, y = Mean, fill = Group)) +
  
  # 1. Bar: 寬度 0.7, 佔位 0.9
  geom_bar(stat="identity", width = bar_w, position = position_dodge(width = dodge_w)) +
  
  # 2. Error Bar: 必須跟著佔位 0.9，寬度設 0.2 較精緻
  geom_errorbar(aes(ymin=Mean-SE, ymax=Mean+SE), 
                width = 0.2, position = position_dodge(width = dodge_w)) +
  
  # 3. 星星: 必須跟著佔位 0.9，Y 軸位置動態計算
  geom_text(aes(y = Mean + SE + star_gap, label = Star),
            position = position_dodge(width = dodge_w), vjust = 0.5) +
  
  # 4. Y 軸: 底部貼齊，頂部留白
  scale_y_continuous(expand = expansion(mult = c(0, 0.15))) +
  
  # 5. 顏色與主題
  scale_fill_manual(values = c("#31688E", "#E67E22")) +
  theme_classic(base_size = 14) # 基礎字體大小 14

# 6. 輸出
ggsave("Final.png", width = 8, height = 10, units = "cm", dpi = 300)




### 第一部分：三大通用核心 (Common Core Logic)

_這部分適用於所有圖表（趨勢圖、Bar 圖、散點圖）。_

#### 1. 空間控制：呼吸感與比例 (Spacing & Scale)

這是你最在意的「圖表胖瘦」與「數據發散」控制中心。

- **Y 軸天花板 (Limits)**：
    
    - **問題**：數據太散，或是星星被切掉。
        
    - **邏輯**：$\text{上限} = \text{最大值} + (\text{全距} \times \text{Buffer係數})$。
        
    - **設定**：`scale_y_continuous(limits = c(min, max_calculated))`。
        
- **畫布留白 (Expand)**：
    
    - **問題**：點貼在邊框上，或是 X 軸太寬/太窄。
        
    - **邏輯**：`expansion(mult = c(底部%, 頂部%))`。
        
    - **Bar 圖口訣**：**底 0 頂留** (`c(0, 0.2)`) -> 讓柱子站穩地板。
        
    - **趨勢圖口訣**：**上下皆留** (`c(0.05, 0.2)`) -> 讓點懸浮。
        

#### 2. 對齊控制：閃避與重疊 (Dodge & Alignment)

這是解決「Error Bar 漂浮」與「星星歪掉」的關鍵。

- **核心參數**：`position_dodge(width = ...)`
    
- **黃金法則**：**所有同一組的元件，寬度參數必須一致。**
    
    - 若 `geom_bar(position = position_dodge(width = 0.9))`
        
    - 則 `geom_errorbar(..., position = position_dodge(width = 0.9))`
        
    - 且 `geom_text(..., position = position_dodge(width = 0.9))`
        
    - _只要這三個數字一樣，它們就會像串燒一樣完美垂直對齊。_
        

#### 3. 外觀控制：精緻度 (Aesthetics)

- **Error Bar**：`width = 0.1` ~ `0.2` (越小越精緻)。
    
- **字體**：`theme(text = element_text(size = 14))` (統一大字體)。
    
- **輸出**：`ggsave(..., units = "cm")` (用純量鎖定物理尺寸)。

### 第二部分：三種場景應用 (Scenario Specifics)

_針對你的不同圖表，這些參數需要「客製化」調整。_

#### 場景 A：兩點趨勢圖 (Pre-Post Trend)

- **目標視覺**：直式長方形 (5:4)，強調前後變化。
    
- **X 軸策略**：**「往內擠壓」**。
    
    - 因為只有兩個點，若不擠壓，線會拉很長。
        
    - `scale_x_discrete(expand = expansion(mult = c(0.35, 0.35)))`
        
    - _解釋_：左右各塞 35% 的空氣，強迫數據集中在中間 30%。
        
- **輸出尺寸**：`width = 4, height = 5` (英寸)。
    

#### 場景 B：三點趨勢圖 (Int-Pre-Post Trend)

- **目標視覺**：正方形 (1:1)，展示完整軌跡。
    
- **X 軸策略**：**「平均分佈」**。
    
    - 三個點需要空間，不能擠。
        
    - `scale_x_discrete(expand = expansion(mult = c(0.1, 0.1)))`
        
    - _解釋_：左右只留 10% 空氣，讓三個點舒適地分佈在 80% 的空間。
        
- **輸出尺寸**：`width = 5, height = 5` (英寸)。
    

#### 場景 C：差值 Bar 圖 (Difference Bar)

- **目標視覺**：強調正負變化與差異。
    
- **基準線**：
    
    - `geom_hline(yintercept = 0, linetype = "dashed")`。
        
    - _技巧_：放在 `geom_bar` 之前畫，讓它成為背景。
        
- **顯著標記**：
    
    - 不使用懸浮星星，改用 **「ㄇ型支架」 (Bracket)**。
        
    - 使用 `geom_segment` 畫橫線，再放星星。

### 第三部分：練習與檢查清單 (Checklist)

下次畫圖時，你可以依照這個順序自我檢查：

1. **[版面]** 這張圖只有兩點嗎？是的話，X 軸 `expand` 設大一點 (`0.35`) 把它擠中間。
    
2. **[Y軸]** 我的星星會被切掉嗎？檢查 `scale_y_continuous` 的上限是否包含 `max * 1.2`。
    
3. **[對齊]** 我的 Error Bar 是歪的嗎？檢查 `geom_bar` 和 `geom_errorbar` 的 `dodge width` 是否相等。
    
4. **[標記]** 我的星星太貼近線了嗎？檢查 `y` 座標是否加了 `buffer` (例如 `mean + se + (range * 0.05)`)。
    
5. **[輸出]** 字體太小嗎？檢查 `ggsave` 的 `width` 是否設太寬，或者 `theme` 的 `base_size` 是否足夠。
